# Governance Rules - GCC Insurance Intelligence Lab

## Mission Statement
Ensure all applications demonstrate fictional insurance logic using synthetic data only. No outputs shall be used for actual underwriting, pricing, reserving, claim approval, or policy issuance.

## Core Principles

### 1. Synthetic Data Only
- **Requirement**: All data must be 100% fabricated
- **Prohibition**: No real insurance data, no real customer information
- **Verification**: All datasets must include clear synthetic markers
- **Penalty**: Violation results in immediate repository suspension

### 2. Human-in-the-Loop Requirement
- **Requirement**: All applications must enforce human review
- **Implementation**: Clear warnings and notices in UI
- **Validation**: Applications must require human confirmation
- **Monitoring**: Regular audits for compliance

### 3. Educational Use Only
- **Purpose**: Demonstration and learning only
- **Restriction**: No production use allowed
- **Notice**: Clear disclaimers in all documentation
- **Compliance**: Regular governance reviews

## Mandatory Requirements

### Documentation Requirements
Every repository must include:

1. **README.md** with:
   - Clear title and description
   - Purpose statement
   - Features list
   - Usage instructions
   - Data sources specification
   - Technical details
   - Critical disclaimer section

2. **model_card.md** with:
   - Model description
   - Intended use cases
   - Limitations and risks
   - Training data specification
   - Ethical considerations
   - Governance requirements

3. **Disclaimer Enforcement**:
   - Top banner in UI
   - Multiple notices throughout
   - Clear production use prohibition
   - Human review requirement

### Code Requirements
- Rule-based logic only (no ML in core functionality)
- Transparent calculations
- Explainable outputs
- Clear audit trails
- Synthetic data validation

### Safety Measures
- No automated decisions
- No financial calculations
- No policy binding
- No premium calculations
- No real-time processing

## Prohibited Activities

### Strictly Forbidden
- Real insurance data usage
- Automated decision making
- Premium or pricing calculations
- Policy binding or issuance
- Customer data processing
- Financial obligation creation
- Production system deployment
- Confidential business logic

### Penalties for Violations
- Repository suspension
- Code review requirement
- Governance training
- Compliance verification
- Possible permanent ban

### Manual Reviews

## Implementation Guidelines

### Required Disclaimers
All applications must include this disclaimer prominently:

```
⚠️ CRITICAL DISCLAIMER

This application demonstrates fictional insurance logic using synthetic data only.

NOT FOR REAL USE:
❌ Actual underwriting decisions
❌ Premium pricing or calculation
❌ Policy binding or issuance
❌ Claims processing or approval
❌ Production insurance operations

FOR EDUCATIONAL PURPOSES ONLY:
✅ Training insurance professionals
✅ Demonstrating insurance concepts
✅ Prototyping insurance workflows
✅ Educational sandbox for GCC markets

All outputs require human review and validation by qualified professionals.
Human-in-the-loop is mandatory for all insurance decisions.
```

### Human-in-the-Loop Enforcement
- Mandatory review notices
- Confirmation requirements
- Clear decision authority limits
- Audit trail maintenance

### Synthetic Data Markers
- All datasets must be clearly labeled
- No real data indicators
- Fabricated scenario indicators
- Educational use clear labeling
